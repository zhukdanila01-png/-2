# Проект: Графовая база данных «Букмекерская контора» (BookmakerDB)

Данный репозиторий содержит материалы лабораторной работы по проектированию, реализации и анализу графовой базы данных в среде **MS SQL Server 2019**, а также её визуализации.

## 📌 Описание проекта
Проект моделирует работу букмекерской конторы с использованием графовых расширений SQL Server (`NODE` и `EDGE`). Система отслеживает пользователей (игроков), спортивные матчи, букмекерские исходы, а также социальные связи между игроками для анализа копирования ставок (капперские синдикаты).

---

## 📂 Структура репозитория
* `script.sql` — SQL-скрипт для создания БД, заполнения данными и выполнения запросов.
* `Bookmaker_Graph.drawio` — двухстраничная диаграмма структуры и данных (Draw.io).
* `Bookmaker_Analytics.pbix` — интерактивный отчет и визуализация графа в Power BI.
* `README.md` — текущее описание проекта.

---

## 🛠️ Реализация в MS SQL Server 2019

### 1. Архитектура графа
* **Узлы (Nodes):**
  * `Player` — пользователи системы (имя, баланс).
  * `Match` — спортивные события (команды, вид спорта, дата).
  * `Odds` — букмекерские исходы и коэффициенты.
* **Рёбра (Edges):**
  * `PlacedBet` — связь игрока с исходом (сумма, дата).
  * `BelongsTo` — связь исхода с конкретным матчем.
  * `FollowsPlayer` — подписка одного игрока на другого (капперская сеть).

### 2. Ключевые аналитические запросы

В проекте реализованы запросы с использованием конструкции `MATCH` и `SHORTEST_PATH`, адаптированные под синтаксис SQL Server 2019:
* Поиск игроков, выбравших определенные матчи.
* Агрегация сумм ставок по видам спорта.
* Расчет цепочек социального влияния капперов (шаблоны `+` и `{1,n}`).
* Поиск скрытых лидеров синдикатов через изоляцию путей графа.

## 📊 Визуализация

### 1. Логическая и физическая схема (Draw.io)
В файле `Bookmaker_Graph.drawio` на отдельных страницах спроектированы:
* **Страница 1:** Мета-модель графа (схемы ограничений соединений `CONNECTION`).
* **Страница 2:** Физический фрагмент графа (конкретные экземпляры игроков, ставок и матчей).

### 2. Интерактивный граф в Power BI
С помощью кастомного визуального элемента **Force-Directed Graph** от Microsoft реализована живая модель графа:
* Настроено SQL-представление `v_GraphEdgesForPowerBI` для импорта данных в формате `Source-Target`.
* Реализовано цветовое разделение типов связей (Ставки, Подписки, Структура матчей).
* Толщина ребер (`Weight`) динамически изменяется в зависимости от суммы сделанной ставки.

> **Результат визуализации:** Граф наглядно подсвечивает цепочки подписок игроков (слева) и их целевую активность на букмекерском рынке (справа).

## 🚀 Как запустить проект локально

1. **Развертывание БД:** Откройте файл `script.sql` в MS SQL Server Management Studio 2019 и выполните его (кнопка *Execute*).
2. **Просмотр схем:**
   Зайдите на сайт [diagrams.net](https://app.diagrams.net/) и откройте файл `Bookmaker_Graph.drawio`.
3. **Аналитика:**
   Установите Power BI Desktop, откройте файл `Bookmaker_Analytics.pbix` и обновите данные, указав строку подключения к вашему локальному серверу SQL Server.
